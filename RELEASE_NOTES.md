# CPA-Helper 0.3.29

本次发布将渠道状态请求条改为最近一小时的真实时间分布，避免七天前的失败请求让当前渠道持续显示为红色。

## 最近请求时间轴

- 最近一小时拆分为 36 个连续时间桶，每个时间桶约 100 秒。
- 时间桶内没有请求显示灰色。
- 请求全部成功显示绿色，请求全部失败显示红色。
- 同一时间桶同时存在成功和失败请求时显示黄色。
- 标题展示最近一小时的实际成功和失败请求数。
- 每根条可查看对应时间桶的成功和失败数量。

## 七天汇总

- 七天请求总数、成功率、费用和账号状态统计保持不变。
- 条形图不再使用七天累计成功率生成伪时间分布。

## 数据库迁移

- 渠道状态快照增加最近窗口起止时间和 36 个时间桶数据。
- 升级启动时自动执行迁移，不需要手工修改数据库。

## 验证

- 后端：`go test ./... -count=1`
- 前端：`npm.cmd run build`
- 覆盖最近时间桶统计、七天窗口隔离、数据库快照持久化和迁移字段测试。
