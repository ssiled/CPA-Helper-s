# CPA-Helper 0.3.24

本次发布修复动态类型号池显示有账号、运行时却返回 503 无候选账号的问题，并强化号池权限与模型目录同步。

## 修复

- 将 `k12`、`team`、`plus` 等动态类型解析为实际 CPA 认证文件 ID，并持久化、同步到 `cpa-auth-pool` 插件。
- 普通用户只能查看和绑定管理员明确授权的号池，避免绑定到未授权的付费号池。
- 新建或更新号池前预检模型目录；部分账号模型获取失败时保留 last-good 快照，不再用不完整结果覆盖运行状态。
- 绑定号池时验证号池存在且启用；本地快照缺失时从插件刷新一次再判断。

## 管理界面

- 号池编辑器新增授权用户多选项，空列表表示仅管理员可绑定。
- 号池状态返回解析后的账号 ID，便于排查动态成员同步结果。

## 构建

- GitHub Actions 在构建和推送镜像前运行后端全量测试与前端生产构建。

## 验证

- 后端：`go test ./...`
- 前端：`npm run build`
