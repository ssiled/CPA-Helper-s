# CPA-Helper 0.3.27

本次发布优化渠道状态快照刷新，解决使用记录较多时 `cpa-helper` 周期性占用单核 CPU 的问题。

## 性能优化

- 定时刷新、Codex Keeper 完成事件和配置变更事件统一进入串行刷新队列。
- 重复刷新请求自动合并，避免多个七天窗口统计任务同时运行。
- 号池用量统计从每个号池遍历全部记录改为全部记录单次扫描。
- 为账号标识建立反向号池索引，同一条记录命中同一号池时只统计一次。
- 渠道状态查询只读取匹配和费用计算需要的字段，减少 SQLite 扫描与 Go 对象分配。
- `raw_json` 只解析一次，并且仅解码账号标识字段，不再重复解码完整请求日志。
- 长时间统计任务会响应刷新超时和进程停止信号。

## 兼容性

- 不包含数据库迁移。
- 不改变渠道状态 API 响应结构。
- 不需要同步升级 CLIProxyAPI 或 CPA Auth Pool Plugin。

## 验证

- 后端：`go test ./... -count=1`
- 渠道状态并发和匹配回归测试连续执行 20 次通过。
- Windows amd64 与 Linux amd64 构建通过。
