# CPA-Helper 0.3.28

本次发布修复渠道状态页面的七天请求分布条未显示失败请求的问题。

## 渠道状态展示

- 请求分布条改为按成功和失败比例映射到固定数量的可视化分段。
- 窗口内只要存在失败请求，至少显示一根红色失败条，避免低失败率被取整隐藏。
- 失败条改为与成功条等高的红色样式，提高辨识度。
- 分布标题直接展示失败请求数量。
- 鼠标悬停时展示最近七天实际成功和失败数量。

## 根因

- 旧实现使用真实请求总数计算失败位置，但页面最多只渲染前 36 根条。
- 请求量较大时，失败条被排在未渲染的位置，因此即使存在失败请求也会显示为全绿色。

## 兼容性

- 不包含数据库迁移或 API 变更。
- 不需要同步升级 CLIProxyAPI 或 CPA Auth Pool Plugin。

## 验证

- 前端：`npm.cmd run build`
